Agent Skills · v1.12.0

Reusable instruction packs. Not prompts.

  • SKILL.md — YAML frontmatter + Markdown body. Version-controlled, hand-written, tool-aware.

  • Different from coding-agent skills in Module 01 — these run inside your agent at runtime.

Unified Memory · PR #4420

Short + Long + Entity, collapsed into one API.

Every write produces a MemoryRecord with content, scope, categories, importance, and embedding. Recall ranks with a weighted composite of semantic similarity, recency decay, and importance.

memory.scope(path)
Bind to a sub-tree. Reads and writes are confined beneath that prefix.
memory.slice([…])
Read-only union of scopes. Great for cross-tenant analytics.
root_scope=
Construction-time prefix. Tenant isolation by construction.
04 Unified Memory 09 / 12

Two recall paths, same signature.

memory.recall(query, depth=…) — pick the cost profile that fits your path.

depth="shallow"
Hot-path retrieval.
One embed, one vector search. Returns MemoryMatches scored by the composite formula. Deterministic, no LLM.
  • embed query
  • single backend search
  • composite rank
depth="deep" (default)
LLM-in-the-loop retriever.
Analyzes the query, generates sub-queries, searches scopes in parallel, routes on confidence, optionally explores deeper to fill evidence gaps.
  • analyze → sub-queries
  • parallel scope search
  • confidence router
  • explore_deeper loop

Checkpointing · v1.14.0–1.14.3

Snapshot the runtime. Rewind anywhere.

  • Works on Flow, Crew, and — new — Agent.kickoff().
  • ~100 event types trigger a write. Signals too — Ctrl-C snapshots before exit.
  • JsonProvider for dev, SqliteProvider for scale. Same six-method contract.
  • Every row carries parent_id + branch — full lineage trees for free.
05 Checkpointing 11 / 12

Rewind. Override. Branch.

Resume rehydrates state and continues. Fork adds a branch label and lets you override any input before the kickoff continues.

# Resume — rehydrate + continue on the same branch crew = Crew.from_checkpoint(cfg) crew.kickoff() # Fork — diverge on a new branch, override inputs crew = Crew.fork(cfg, branch="aggressive") crew.kickoff(inputs={"tone": "aggressive and irreverent"}) # Or from the CLI — no code required $ crewai checkpoint resume <ID>
gotcha SqliteProvider wants the "db_path#id" form on restore_from.
