Unified Memory · PR #4420

Memory isn't storage. It's cognition.

Most systems treat memory as a vector store plus retrieval — text in, text out. Here, memory reasons about what to keep, when to merge, and when to let go.

“Memory itself resembles an Agentic System.” — CrewAI engineering blog
04 Unified Memory 09 / 12

Two ways to treat memory. One is thinking.

Traditional RAG dumps text into a vector store and stuffs the top-k into context. Cognitive memory reasons about what to keep, when to merge, and when to let go.

Memory as Storage
Linear. Mechanical.
  1. Data in
  2. Embed
  3. Store
  4. Retrieve
  5. Stuff into context
Can't tell what matters from what doesn't. Can't update itself. Ages badly.
Memory as Cognition
Cyclical. Self-aware.
  • Encodeanalyzes, places, scores
  • Consolidatedetects + resolves conflicts
  • Recallevaluates its own confidence
  • Extractatomizes long text into facts
  • Forgetprunes intentionally
Every op is a Flow. The system reasons about what to keep.

Checkpointing · v1.14.0–1.14.3

Snapshot the runtime. Replay from any step.

  • Works on Flow, Crew, and — new — Agent.kickoff().
  • ~100 event types trigger a write. Signals too — Ctrl-C snapshots before exit.
  • JsonProvider for dev, SqliteProvider for scale. Same six-method contract.
  • Every row carries parent_id + branch — full lineage trees for free.
05 Checkpointing 11 / 12

Three methods. Pick the shape.

Same CheckpointConfig. Every Flow, Crew, and Agent has all three.

.kickoff(from_checkpoint=cfg)
Start a run with checkpointing on. If cfg.restore_from is set, it's a resume instead of a fresh run.
Use when Default entry. One method for both "turn it on" and "pick up where you left off."
.from_checkpoint(cfg)
Restore and return the rehydrated instance on the same branch. Same inputs. Completed steps kept.
Use when You want the restored entity back — to inspect state, or to mutate fields before kickoff.
.fork(cfg, branch="…")
Restore AND tag a new branch. New checkpoints land on their own lineage; parent_id points at the source.
Use when Experimenting — try a different input from the same point without clobbering main.
gotcha After from_checkpoint / fork, wire instance.checkpoint = cfg.model_copy(update={"restore_from": None}) so the resumed run keeps writing new checkpoints.
